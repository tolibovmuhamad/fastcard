Ты сохраняешь работу по проекту **fastcard** в Git и отправляешь на GitHub. Общайся по-русски.
Выполняй шаги по порядку, используя PowerShell. После каждого ключевого шага проверяй результат.

## 1. Убедись, что это git-репозиторий
- Проверь: `git rev-parse --is-inside-work-tree`.
- Если репозитория нет — инициализируй: `git init`, затем `git branch -M main`.
- Убедись, что есть `.gitignore` (если нет — создай с `node_modules/`, `dist/`, `.env*`, и т.п.).

## 2. Подготовь сообщение коммита
- Если в `$ARGUMENTS` передан текст — используй его как сообщение коммита.
- Иначе прочитай `.claude/sessions/LAST_SESSION.md` и собери сообщение вида:
  `Сессия №<N>: <1–2 главных итога>`.

## 3. Сделай коммит
- `git add -A`
- Проверь, есть ли что коммитить: `git status --porcelain`. Если пусто — сообщи «нет изменений» и остановись.
- Коммить с сообщением. В конце сообщения коммита добавь строку:
  `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`

## 4. Запушь в GitHub
- Проверь удалённый репозиторий: `git remote get-url origin`.
- **Если `origin` есть:** `git push -u origin main` (при первом пуше с `-u`, дальше просто `git push`).
- **Если `origin` НЕТ** — пуш невозможен без подключения GitHub. Сообщи пользователю и предложи
  ОДИН из вариантов (не выполняй сам без его данных):
  1. Установить GitHub CLI и создать репозиторий: `gh repo create fastcard --private --source=. --push`
     (после `gh auth login`).
  2. Создать пустой репозиторий на github.com вручную и дать ссылку, тогда выполнить:
     `git remote add origin <URL>` и `git push -u origin main`.

## 5. Отчёт
Кратко на русском: что закоммичено (хэш и сообщение) и запушено ли в GitHub. Если пуш не сделан —
напомни, что нужно для подключения GitHub.
