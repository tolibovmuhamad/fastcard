## 1. Общие сведения

**Название проекта:** fastcard

**Назначение:** веб-приложение интернет-магазина с витриной для покупателей и административной панелью для управления каталогом и заказами.

**Краткое описание:** пользователь просматривает каталог товаров, добавляет их в корзину и избранное, оформляет заказ и отслеживает его статус в личном кабинете. Администратор управляет товарами, категориями и заказами через защищённую панель. Авторизация и разграничение доступа — по JWT-токену.

**Объём:** 12 уникальных экранов витрины (по макету Figma) + админ-панель (товары, заказы, категории; дизайн в макете отсутствует).

## 5. Архитектура и структура проекта

Единое React-приложение. Витрина и админка разграничены маршрутами (`/` — витрина, `/admin/*` — панель администратора). Отдельные приложения не создаются.

Рекомендуемая структура каталогов:

```
src/
├── app/            # точка входа, провайдеры, корневой роутинг
├── pages/          # страницы (по одной на маршрут)
├── components/     # переиспользуемые UI-компоненты
├── features/       # модули по доменам (cart, auth, products, orders…)
├── api/            # axios-инстанс, интерсепторы, запросы к эндпоинтам
├── store/          # сторы Zustand (auth, cart, …)
├── types/          # TypeScript-типы (модели из Swagger)
├── lib/            # вспомогательные функции
└── routes/         # описание маршрутов и защищённые роуты (guards)
```

## 8. Авторизация и безопасность (JWT)

- При входе/регистрации сервер возвращает JWT. Токен сохраняется на клиенте (способ хранения — localStorage; при необходимости пересмотреть в сторону более защищённого варианта).
- axios-интерсептор добавляет заголовок `Authorization: Bearer <token>` ко всем защищённым запросам.
- При ответе `401 Unauthorized`:
  - если в API есть refresh-токен — попытка обновления токена;
  - иначе — выход пользователя и редирект на страницу входа.
- Защищённые маршруты (guards):
  - маршруты покупателя — требуют авторизации;
  - маршруты `/admin/*` — требуют роли «администратор».
- Хранение текущего пользователя и статуса авторизации — в сторе Zustand.

> Конкретный формат токена, наличие refresh-механизма и поле роли — **сверить со Swagger**.

## 9. Управление состоянием (Zustand)

| Стор | Назначение |
|------|-----------|
| authStore | текущий пользователь, токен, статус авторизации, методы входа/выхода |
| cartStore | содержимое корзины, добавление/удаление, пересчёт суммы, синхронизация с localStorage |
| favoritesStore | список избранного |
| (по необходимости) uiStore | состояние интерфейса (модалки, уведомления) |

Данные с сервера (товары, заказы, категории) запрашиваются через axios. Кэширование — по необходимости.

## 10. Работа с API

Базовый адрес и точные эндпоинты — по Swagger (https://store-api.softclub.tj). Ниже — **предполагаемая** группировка ресурсов для интернет-магазина. Точные пути, методы, параметры и модели **обязательно сверить со Swagger** перед реализацией.

| Группа | Назначение (предположительно) |

**Порядок:** после открытия Swagger в браузере зафиксировать реальные эндпоинты и модели, описать TypeScript-типы в `src/types/`, после чего реализовать запросы в `src/api/`.
