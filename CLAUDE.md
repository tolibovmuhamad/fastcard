This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Автоматизация сессий (команды /start и /stop)

В проекте настроена файловая «память» для работы между сессиями:
- **`ROADMAP.md`** — главный план работ (этапы и задачи со статусами `[ ] [~] [x] [!]`).
- **`.claude/sessions/LAST_SESSION.md`** — итоги последней сессии (напоминание «где остановились»).
- **`.claude/sessions/SESSIONS.md`** — журнал всех сессий.
- **`.claude/commands/start.md`** — слэш-команда `/start`: читает `ROADMAP.md` и `LAST_SESSION.md`,
  показывает прогресс и следующий шаг, начинает работу.
- **`.claude/commands/stop.md`** — слэш-команда `/stop`: обновляет статусы в `ROADMAP.md`, перезаписывает
  `LAST_SESSION.md` и дописывает запись в `SESSIONS.md`.

Рабочий цикл: пользователь пишет `/start` в начале и `/stop` в конце сессии.

## Статус проекта

Проект **инициализирован** (Этап 1.1): Vite 8 + React 19 + TypeScript 6. Единое название —
**`fastcart`** (как в логотипе макета). Источник истины по объёму — `TZ_fastcard.md`; результаты
сверки со Swagger и Figma — в `docs/SOURCES_NOTES.md`.

**Команды:**
- `npm install` — установка зависимостей
- `npm run dev` — дев-сервер (http://localhost:5173)
- `npm run build` — прод-сборка (`tsc -b && vite build` → `dist/`)
- `npm run preview` — предпросмотр сборки
- `npm run lint` / `npm run lint:fix` — ESLint (flat config, `eslint.config.js`)
- `npm run format` / `npm run format:check` — Prettier

Алиас импорта: `@/*` → `src/*` (настроен в `vite.config.ts` и `tsconfig.app.json`).

**Вёрстка (Этап 1.2):** Tailwind CSS **v4** (CSS-first, плагин `@tailwindcss/vite`; тема и
CSS-переменные в `src/index.css`) + shadcn/ui (стиль `new-york`, базовый цвет `neutral`,
конфиг `components.json`, утилита `cn` в `src/lib/utils.ts`, компоненты в `src/components/ui/`).
Добавление компонентов: `npx shadcn@latest add <name>`.

**Структура `src/` (Этап 1.3):** `app/ pages/ components/ (components/ui/) features/ api/ store/
types/ lib/ routes/ hooks/`. Пустые папки помечены `.gitkeep`.

**API-слой (Этап 1.5):** `src/api/` — axios-инстанс `apiClient` (`client.ts`) с интерсепторами
(Bearer-токен из `tokenStorage`; при `401` — сброс токена и редирект на `/login`, refresh нет),
базовый адрес из `VITE_API_BASE_URL` (`config.ts`, `.env.example`). Токен — в localStorage
(`tokenStorage.ts`). Логаут через `authStore` подключится на Этапе 2.

**Роутинг (Этап 1.6):** React Router v8, `createBrowserRouter` + `RouterProvider` (в `App.tsx`).
Конфиг — `src/routes/router.tsx`, пути — `src/routes/paths.ts` (`ROUTES`). Страницы (`src/pages/`,
по одной на маршрут) грузятся лениво через `React.lazy` (отдельные чанки); Suspense-фолбэк
`PageLoader` — в лейаутах `src/app/RootLayout.tsx` (витрина) и `src/app/AdminLayout.tsx` (`/admin`).

**Layout (Этап 1.7):** `src/components/layout/` — `Header` (логотип, меню, поиск→каталог, иконки
избранного/корзины/аккаунта; счётчики наполнятся из сторов на Этапах 4–5) и `Footer` (Exclusive +
подписка-заглушка, Support, Account, Quick Link, соц-иконки — инлайн-SVG в `SocialIcons.tsx`, т.к.
lucide больше не поставляет бренд-логотипы). Используются в `RootLayout`. shadcn-компоненты:
`Button`, `Input` (`src/components/ui/`).

**Этап 1 завершён.** Следующий — Этап 2 (авторизация). На интеграции уточнить по факту: форма ответа
`POST /Account/login`, claim роли в JWT, логин по `userName` vs email (см. `docs/SOURCES_NOTES.md` §4).

`TZ_fastcard.md` — главный источник истины по объёму и решениям. Прочитай его перед началом работы.

## Технологический стек (раздел 3 ТЗ)

TypeScript · React · Vite · React Router · Zustand (состояние) · axios (с интерсепторами) ·
shadcn/ui (Tailwind CSS + Radix UI). Авторизация по JWT (access-токен; refresh — только если есть в API).

не проводится.

## Архитектура (раздел 5 ТЗ)

Единое React-приложение. Витрина и админка разделены маршрутами, а не отдельными приложениями:
`/` — витрина, `/admin/*` — админ-панель. Рекомендуемая структура:

```
src/
├── app/         # точка входа, провайдеры, корневой роутинг
├── pages/       # страницы (по одной на маршрут)
├── components/  # переиспользуемые UI-компоненты
├── features/    # модули по доменам (cart, auth, products, orders, …)
├── api/         # axios-инстанс, интерсепторы, запросы к эндпоинтам
├── store/       # сторы Zustand (auth, cart, …)
├── types/       # TypeScript-типы (модели из Swagger)
├── lib/         # вспомогательные функции
└── routes/      # описание маршрутов и защищённые роуты (guards)
```

### Авторизация и доступ (раздел 8 ТЗ)
- Три роли: **гость** (просмотр, поиск, корзина — но оформление требует входа), **покупатель**
  (оформление заказа, кабинет, история заказов, избранное), **администратор** (`/admin/*`).
- Роль читается из payload JWT (название поля — **сверить со Swagger**).
- axios-интерсептор добавляет `Authorization: Bearer <token>` к защищённым запросам.
- При `401`: если API поддерживает refresh — обновить токен; иначе — выход и редирект на вход.
- Guards: маршруты покупателя требуют авторизации; `/admin/*` требует роль администратора.
- Текущий пользователь и статус авторизации — в сторе `authStore` (Zustand).

### Состояние (Zustand, раздел 9 ТЗ)
- `authStore` — пользователь, токен, статус авторизации, вход/выход.
- `cartStore` — содержимое корзины, добавление/удаление, пересчёт сумм, **сохранение в localStorage**
  (корзина переживает сессии).
- `favoritesStore` — избранное.
- по необходимости `uiStore` — модалки, уведомления.
Данные с сервера (товары, заказы, категории) запрашиваются через axios; кэширование — по необходимости.

## Порядок разработки (раздел 13 ТЗ)

1. Настройка проекта (Vite + React + TS, Tailwind + shadcn/ui, структура папок, axios-инстанс, базовый роутинг)
2. Авторизация (вход, регистрация, хранение JWT, интерсепторы, guards, `authStore`)
3. Витрина — просмотр (Главная, Каталог с фильтрами/сортировкой, Карточка товара)
4. Корзина и оформление (`cartStore`, корзина, checkout с заглушкой оплаты, экран успеха)
5. Кабинет (профиль, мои заказы, избранное)
6. Админка (товары, заказы, категории)
7. Финализация (адаптив, обработка ошибок и загрузки, отладка)
